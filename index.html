Library

UI-библиотека для Roblox-исполнителей (executors): окна, страницы, секции, тогглы, слайдеры, дропдауны, колорпикеры, кейбинды, конфиги и полная темизация. Один файл, без зависимостей, кроме кастомного шрифта.

Быстрый старт → Открыть полный пример

Введение

Library — самописная GUI-библиотека на Luau, рассчитанная на запуск через скрипт-executor (loadstring / getgenv). Она строит интерфейс из вложенных сущностей:

Window

Главное окно приложения. Перетаскиваемое и изменяемое по размеру.

Page

Вкладка в левом сайдбаре окна, со своей иконкой.

SubPage

Под-вкладка внутри Page, делит контент на колонки (Side 1 / 2).

Section

Карточка-контейнер с заголовком, в которую кладутся элементы управления.

Все элементы управления (Toggle, Slider, Dropdown и т.д.) создаются методами на объекте Section, а их состояние хранится в глобальной таблице Library.Flags по ключу Flag.

Важно: библиотека работает только в среде Roblox executor — ей нужны функции getgenv, writefile, isfile, makefolder, getcustomasset и т.д. Обычный Roblox Studio / стандартный клиент их не предоставляет.

Установка

Загрузите файл библиотеки через ваш executor. Если Library уже существует в getgenv(), она автоматически выгружается перед повторной загрузкой — это позволяет безопасно перезапускать скрипт.

Copylocal Library = loadstring(game:HttpGet("https://.../library.lua"))()

Внутри файла уже встроена защита от повторной загрузки:

Copyif getgenv().Library then
    getgenv().Library:Unload()
end

Быстрый старт

Минимальный рабочий пример: окно → страница → подстраница → секция → тоггл.

Copylocal Window = Library:Window({Name = "my hub"})

local Page = Window:Page({Name = "Main", Icon = "136879043989014"})
local SubPage = Page:SubPage({Name = "Combat"})
local Section = SubPage:Section({Name = "Aimbot", Side = 1})

Section:Toggle({
    Name = "Enabled",
    Flag = "AimbotEnabled",
    Default = false,
    Callback = function(Value)
        print("Aimbot:", Value)
    end
})
Подсказка: клавиша по умолчанию для открытия/закрытия окна — Z (Library.MenuKeybind). Меняется на странице настроек или напрямую через Library.MenuKeybind.

Window

Создаёт главное окно: перетаскиваемый фрейм с заголовком, иконкой первой буквы имени, сайдбаром страниц и блоком статуса подписки/сессии.

методLibrary:Window(Data)#

Возвращает объект Window. Автоматически центрируется и открывается с анимацией появления.

ПолеТипПо умолчаниюОписание
Namestring"Window"Заголовок окна. Первая буква используется как значок-аватар.
SubTitlestring"he"Подзаголовок под именем (например, название игры).
ExpiresInstring"23d"Текст в нижнем блоке: "Sub expires in …".
Copylocal Window = Library:Window({
    Name = "ui for skidhub",
    SubTitle = "Blox Fruits",
    ExpiresIn = "30d"
})

методWindow:SetOpen(Bool)#

Плавно показывает или скрывает окно (фейд всех потомков по transparency-свойствам). Привязано к Library.MenuKeybind по умолчанию.

методWindow:SetCenter()#

Пересчитывает позицию окна из AnchorPoint(0.5,0.5) в абсолютные координаты — вызывается автоматически при создании, чтобы перетаскивание/ресайз работали корректно.

Category

методLibrary:Category(Name)#

Добавляет текстовый разделитель-заголовок в список страниц сайдбара (например, "Settings" перед страницей настроек). Чисто декоративный элемент, ничего не возвращает для дальнейшей вложенности.

Делает страницу активной/неактивной (переключение вызывается автоматически кликом по вкладке в сайдбаре, но можно вызвать вручную).

SubPage

Горизонтальные под-вкладки внутри Page (отображаются под заголовком страницы). Каждая SubPage делит свою область контента на колонки.

методPage:SubPage(Data)#

ПолеТипПо умолчаниюОписание
Namestring"SubPage"Текст под-вкладки.
Columnsnumber2Кол-во вертикальных колонок (используются как Side в Section).
Copylocal CombatSub = CombatPage:SubPage({Name = "Combat", Columns = 2})
CombatPage:SubPage({Name = "Weapon"})
CombatPage:SubPage({Name = "FoV"})
Первая созданная SubPage внутри Page становится активной по умолчанию.

Section

Карточка с заголовком и (опционально) иконкой. Всё содержимое — тогглы, кнопки, слайдеры и т.д. — добавляется внутрь неё. Каждая Section привязана к одной из колонок SubPage через Side.

методSubPage:Section(Data)#

ПолеТипПо умолчаниюОписание
Namestring"Section"Заголовок карточки.
Iconstring""ID ассета иконки рядом с заголовком.
Sidenumber1Номер колонки SubPage, в которую помещается секция (1 = левая, 2 = правая и т.д.).

local Right = CombatSub:Section({Name = "Visuals", Side = 2})

Slider

методSection:Slider(Data)#

Горизонтальный слайдер с перетаскиваемым ползунком. Реагирует на клик в любую точку полосы и на драг.

ПолеТипПо умолчаниюОписание
Namestring"Slider"Подпись.

Maxnumber100Максимум диапазона.

Dropdown

методSection:Dropdown(Data)#

Выпадающий список с возможностью одиночного или множественного выбора. Открывается во всплывающем окне поверх интерфейса.

ПолеТипПо умолчаниюОписание
Namestring"Dropdown"Подпись слева.
FlagstringавтоКлюч в Flags: string (одиночный режим) или table (Multi).
Itemstable<string>{}Список опций при создании.
Defaultstring / table""Опция(и), выбранные по умолчанию.
MultibooleanfalseРазрешить выбор нескольких опций одновременно.
Callbackfunction(value)noopВызывается при любом изменении выбора.
Copylocal Weapon = Section:Dropdown({
    Name = "Weapon",
    Flag = "Weapon",
    Items = {"Sword", "Gun", "Fists"},
    Default = "Sword",
    Multi = false,
    Callback = function(Value)
        print("Selected:", Value)
    end
})

методDropdown:Get()#

Возвращает текущее значение (строка или таблица строк в режиме Multi).

методDropdown:Set(Option)#

Программно выбирает опцию(и). В Multi-режиме принимает таблицу строк.

методDropdown:Add(Option)#

Добавляет новую опцию в список во время выполнения.

методDropdown:Remove(Option)#

Удаляет опцию по имени.

методDropdown:Refresh(List)#

Полностью пересоздаёт список опций из переданной таблицы строк (используется, например, для списка конфигов).

CopyWeapon:Add("Bow")
Weapon:Remove("Fists")
Weapon:Refresh({"Sword", "Bow", "Staff"})

Keybind

Кнопка для назначения клавиши/кнопки мыши, с тремя режимами срабатывания. Создаётся через :Keybind() на Toggle или Label.

методToggle:Keybind(Data) / Label:Keybind(Data)#

ПолеТипПо умолчаниюОписание
FlagstringавтоКлюч в Flags. Хранит {Key, Mode, Toggled}.
DefaultEnum.KeyCodeEnum.KeyCode.EКлавиша по умолчанию.
Mode"Toggle" | "Hold" | "Always""Toggle"Логика срабатывания (см. ниже).
Callbackfunction(toggled: boolean)noopВызывается при срабатывании/отпускании.

Toggle

Каждое нажатие инвертирует состояние.

Hold

Активно только пока клавиша зажата.

Always

Всегда true вне зависимости от клавиши (полезно для "включено по умолчанию").

CopySection:Label("Aimbot Key"):Keybind({
    Flag = "AimKey",
    Default = Enum.KeyCode.Q,
    Mode = "Hold",
    Callback = function(Active)
        print("Aiming:", Active)
    end
})

методKeybind:Get()#

Возвращает Key, Mode, Toggled.

методKeybind:Set(Key)#

Принимает Enum.KeyCode/Enum.UserInputType, таблицу {Key, Mode} или строку режима ("Toggle"/"Hold"/"Always").

методKeybind:Press(Bool)#

Программно эмулирует нажатие/отпускание с учётом текущего режима.

Темизация

Все цвета интерфейса завязаны на таблицу Library.Theme. Любой элемент, помеченный в коде библиотеки через :AddToTheme(), автоматически обновится при смене темы.

Background
Inline
Element
Accent
Border
Border 2

методLibrary:ChangeTheme(Theme, Color)#

Меняет именованный цвет темы (например "Accent") и применяет его ко всем подписанным элементам.

CopyLibrary:ChangeTheme("Accent", Color3.fromRGB(120, 180, 255))

методLibrary:AddToTheme(Item, Properties)#

Подписывает инстанс на тему: Properties — таблица {PropertyName = "ThemeKey"}. Используется внутри самой библиотеки при создании каждого элемента, но доступна и для кастомных инстансов.

CopyMyFrame:AddToTheme({BackgroundColor3 = "Element"})

методLibrary:ChangeItemTheme(Item, Properties)#

Переопределяет, на какие свойства темы реагирует конкретный инстанс (используется, например, в Button при анимации нажатия).

Встроенная страница настроек (CreateSettingsPage) уже содержит вкладку Theming с колорпикером для каждого ключа темы — отдельно реализовывать UI для этого не нужно.

Flags & SetFlags

Library.Flags — единая таблица состояния всех элементов с Flag. Ключ — строка Flag, значение зависит от типа элемента.

ЭлементТип значения в Flags
Toggleboolean
Slidernumber
Dropdownstring | table<string>
Textboxstring
Colorpicker{Color: Color3, HexValue: string}
Keybind{Key: string, Mode: string, Toggled: boolean}
Copyif Library.Flags["AimbotEnabled"] then
    -- логика аимбота
end

Library.SetFlags — таблица функций-сеттеров по тем же ключам, используется внутри LoadConfig для применения сохранённых значений к уже созданным элементам.

Конфиги

Конфиги сохраняются как JSON-файлы в папке Library.Folders.Configs ("Crown/Configs" по умолчанию) через файловое API executor'а.

методLibrary:GetConfig()#

Сериализует текущие Library.Flags в JSON-строку. Keybind и Colorpicker сериализуются в специальном формате ({Key, Mode} / {Color, Alpha}).

методLibrary:LoadConfig(Config)#

Принимает JSON-строку, парсит и применяет значения через Library.SetFlags к уже существующим элементам.

методLibrary:DeleteConfig(Config)#

Удаляет файл конфига по имени (с расширением), если он существует.

методLibrary:RefreshConfigsList(Element)#

Пересчитывает список файлов в папке конфигов и вызывает Element:Refresh(...) на переданном Dropdown.

Copy-- сохранить
writefile(Library.Folders.Configs .. "/profile1.json", Library:GetConfig())

-- загрузить
local Data = readfile(Library.Folders.Configs .. "/profile1.json")
Library:LoadConfig(Data)
Готовый UI для создания/удаления/загрузки/сохранения конфигов уже встроен во вкладку Settings → Configs, создаваемую через CreateSettingsPage.

CreateSettingsPage

методLibrary:CreateSettingsPage(Window)#

Готовая страница настроек "под ключ" — добавляет в окно вкладку Settings с тремя под-вкладками:

Configs

Создание, сохранение, загрузка, удаление и обновление списка конфигов.

Theming

Колорпикер для каждого ключа в Library.Theme.

Settings

Unload-кнопка, кейбинд открытия меню, скорость/стиль/направление твинов.

Copylocal Window = Library:Window({Name = "my hub"})
-- ... создаём остальные страницы ...
local SettingsPage = Library:CreateSettingsPage(Window)

Утилиты

методLibrary:Thread(Function)#

Запускает функцию в отдельной corutine, отслеживаемой библиотекой (закрывается при Unload).

методLibrary:Connect(Event, Callback, Name?)#

Подключает обработчик к RBXScriptSignal с авто-учётом соединения в Library.Connections для последующей отписки.

методLibrary:Disconnect(Name)#

Отписывает именованное соединение, созданное через Connect.

методLibrary:SafeCall(Function, ...)#

Обёртка над pcall: при ошибке логирует через warn и возвращает false вместо краша. Используется при каждом вызове пользовательских Callback.

методLibrary:Round(Number, Float)#

Округляет число с заданным шагом (например, Round(57.3, 0.1) → 57.3).

методLibrary:Lerp(Start, Finish, Time)#

Линейная интерполяция между двумя числами.

методLibrary:ToRich(Text, Color)#

Оборачивает текст в <font color="rgb(...)"> для richtext-лейблов на основе Color3.

методLibrary:IsMouseOverFrame(Frame)#

Проверяет, находится ли курсор мыши внутри границ переданного фрейма. Используется для закрытия попапов по клику вне их.

Unload

методLibrary:Unload()#

Полностью выгружает интерфейс: отключает все соединения, закрывает все потоки, уничтожает корневой ScreenGui и очищает getgenv().Library. Вызывается автоматически при повторной загрузке скрипта, либо вручную (например, кнопкой "Unload" на странице настроек).

CopyLibrary:Unload()

Полный пример

Сборка окна с категориями, несколькими страницами, под-страницами на две колонки и всеми типами элементов — аналогично демо-блоку в исходнике библиотеки.

Copylocal Window = Library:Window({Name = "ui for skidhub"})

Window:Category("Main")
local CombatPage = Window:Page({Name = "Combat", Icon = "136879043989014"})
local VisualsPage = Window:Page({Name = "Visuals", Icon = "136879043989014"})

local CombatSub = CombatPage:SubPage({Name = "Combat"})
CombatPage:SubPage({Name = "Weapon"})

Window:Category("Settings")
local SettingsPage = Library:CreateSettingsPage(Window)

local AimbotSection = CombatSub:Section({Name = "Aimbot", Icon = "136879043989014", Side = 1})

AimbotSection:Toggle({Name = "Enabled", Flag = "AimbotEnabled", Callback = function(v) print(v) end})
AimbotSection:Button({Name = "Reset", Callback = function() print("reset") end})
AimbotSection:Slider({Name = "FOV", Flag = "FOV", Min = 0, Max = 100, Default = 50, Suffix = "%"})
AimbotSection:Dropdown({Name = "Priority", Flag = "Priority", Items = {"Distance", "Health"}, Default = "Distance"})
AimbotSection:Label("FOV Color"):Colorpicker({Flag = "FOVColor", Default = Color3.fromRGB(255,255,255)})
AimbotSection:Label("Aim Key"):Keybind({Flag = "AimKey", Default = Enum.KeyCode.E, Mode = "Hold"})
AimbotSection:Textbox({Flag = "TargetName", Placeholder = "Username"})

Library:Notification("UI loaded", 4, nil)
Library — справочная документация сгенерировано из исходного кода
